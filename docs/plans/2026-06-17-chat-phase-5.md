# Phase 5: Chat 日常问答 — 实施方案

**日期**：2026-06-17  
**阶段**：Phase 5  
**目标**：实现轻量日常问答窗口，支持多会话管理、AI 自动命名、流式输出、Markdown 渲染和动态推荐。

---

## 一、流式方案选型：SSE（推荐）

| 维度 | SSE（fetch + ReadableStream） | WebSocket |
|---|---|---|
| 实现成本 | **低**，与 Translate 模块完全相同的模式，后端用 `SseEmitter` 或 `Flux<ServerSentEvent>` | 高，需引入 Spring WebSocket / STOMP，配置 endpoint、消息代理 |
| 认证 | **原生支持 JWT**，fetch 可携带 Authorization header | 握手阶段 Header 传值受限，需自实现 Token 验证逻辑 |
| 传输方向 | 服务端→客户端单向（完全满足 request-stream 模式） | 全双工，对 Chat 场景属于过度设计 |
| 运维复杂度 | **低**，普通 HTTP，无需代理层特殊配置 | 连接保活、心跳、断线重连均需自行处理 |
| 项目复用 | `translate.api.ts` 中的 `stream()` 已有完整实现，可直接复制改造 | 全新实现 |

**结论：使用 SSE，复用 Translate 的 fetch + ReadableStream 模式。**

## 二、数据库设计（V1_15）

### `chat_conversations` — 对话会话表

```sql
CREATE TABLE chat_conversations (
    id            TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
    user_id       TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    title         TEXT NOT NULL DEFAULT '新对话',
    title_ai      BOOLEAN NOT NULL DEFAULT TRUE,   -- true=AI 生成，false=用户手动修改
    workflow_type TEXT NOT NULL DEFAULT 'chat',
    message_count INT  NOT NULL DEFAULT 0,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_chat_conv_user_updated ON chat_conversations(user_id, updated_at DESC);
```

### `chat_messages` — 消息表

```sql
CREATE TABLE chat_messages (
    id              TEXT PRIMARY KEY DEFAULT gen_random_uuid()::text,
    conversation_id TEXT NOT NULL REFERENCES chat_conversations(id) ON DELETE CASCADE,
    role            TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
    content         TEXT NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_chat_msg_conv_created ON chat_messages(conversation_id, created_at ASC);
```

> 同时在 `V1_15` 迁移末尾插入 `chat` workflow_type：
> ```sql
> INSERT INTO workflow_llm_configs (id, workflow_type)
> VALUES (gen_random_uuid()::text, 'chat');
> ```

### 3.3 ChatService — 核心方法签名

```java
@Service
public class ChatService {
    // 创建新对话（title 为空时后端暂存"新对话"，消息首发后异步触发 AI 命名）
    public ChatConversationResponse createConversation(String userId);

    // 获取对话列表，按 updated_at DESC，分组数据由前端处理
    public List<ChatConversationResponse> listConversations(String userId);

    // 获取指定对话的历史消息（按 created_at ASC）
    public List<ChatMessageResponse> getMessages(String conversationId, String userId);

    // 重命名（含 titleAi 标志）
    public void renameConversation(String id, String userId, ChatRenameRequest req);

    // AI 提炼标题：取最近 N 条消息摘要，调 LLM 生成 ≤ 15 字的标题
    public String generateTitle(String conversationId, String userId);

    // 删除对话（级联删消息）
    public void deleteConversation(String id, String userId);

    // SSE 流式发送：持久化 user 消息 → 构建上下文 → 调 StreamingChatLanguageModel
    // → token 逐个写入 SseEmitter → done 后持久化 assistant 消息 → 触发 AI 命名（若 titleAi=true）
    public SseEmitter sendMessage(String conversationId, String userId, ChatSendRequest req);

    // 动态推荐：取最近 7 天的对话 title，去重截断后返回，缺少时用预置词条兜底
    public List<ChatSuggestionResponse> getSuggestions(String userId);
}
```

### 3.5 SSE 流式实现骨架

```java
@PostMapping(value = "/{id}/messages/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter stream(@PathVariable String id,
                         @RequestBody ChatSendRequest req,
                         @AuthenticationPrincipal UserDetails user) {
    return chatService.sendMessage(id, user.getUsername(), req);
}

// ChatService.sendMessage 内部逻辑：
// 1. 持久化 user 消息，message_count++
// 2. 取最近 N 条历史构建 List<ChatMessage> context
// 3. StreamingChatLanguageModel model = llmConfigService.resolveStreamingModel("chat")
// 4. SseEmitter emitter = new SseEmitter(120_000L)
// 5. model.generate(context, new StreamingResponseHandler<>() {
//        onNext(token)  → emitter.send(SseEmitter.event().name("token").data(token))
//        onComplete(r)  → 持久化 assistant 消息，发 done 事件，triggerAiTitle(conv)
//        onError(e)     → emitter.send(event().name("error")...), emitter.completeWithError(e)
//    })
// 6. return emitter
```

### 3.6 AI 命名策略

- **触发时机**：每次 assistant 消息落库后，若 `titleAi = true`，异步（`@Async`）调用 `generateTitle()`
- **Prompt**：`"根据以下对话内容，用 10 个字以内的中文生成一个准确的对话标题，只输出标题本身：\n{前 3 轮对话摘要}"`
- **防抖**：`message_count <= 2` 时才触发（第一次往返后命名，后续不再覆盖）
- **用户重命名后**：`titleAi = false`，永远不再被 AI 覆盖

### 4.3 chat.api.ts — 关键接口

```typescript
export type ChatStreamEvent = {
  type: 'token' | 'done' | 'error'
  payload: { token?: string; message?: ChatMessage; error?: string } | null
}

export const chatApi = {
  listConversations: () =>
    apiClient.get<ApiResponse<ChatConversation[]>>('/chat/conversations'),

  createConversation: () =>
    apiClient.post<ApiResponse<ChatConversation>>('/chat/conversations'),

  deleteConversation: (id: string) =>
    apiClient.delete<ApiResponse<void>>(`/chat/conversations/${id}`),

  renameConversation: (id: string, title: string, titleAi: boolean) =>
    apiClient.patch<ApiResponse<void>>(`/chat/conversations/${id}/title`, { title, titleAi }),

  generateTitle: (id: string) =>
    apiClient.post<ApiResponse<string>>(`/chat/conversations/${id}/title/ai`),

  getMessages: (id: string) =>
    apiClient.get<ApiResponse<ChatMessage[]>>(`/chat/conversations/${id}/messages`),

  // 与 translateApi.stream 完全相同的 fetch + ReadableStream 模式
  stream: async (
    conversationId: string,
    message: string,
    onEvent: (e: ChatStreamEvent) => void,
  ) => {
    const token = useAuthStore.getState().accessToken
    const res = await fetch(`/api/v1/chat/conversations/${conversationId}/messages/stream`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        ...(token ? { Authorization: `Bearer ${token}` } : {}),
      },
      body: JSON.stringify({ message }),
    })
    // ... 与 translate.api.ts stream() 相同的 ReadableStream 读取逻辑
  },

  getSuggestions: () =>
    apiClient.get<ApiResponse<ChatSuggestion[]>>('/chat/suggestions'),
}
```

## 七、关键约束备忘

- **上下文窗口**：`sendMessage` 时取最近 20 条消息（user + assistant 交替）构建上下文，避免 token 爆炸
- **并发 SSE**：同一对话同时只允许一个 SSE 连接（前端发送前 abort 上一个）
- **AI 命名防抖**：`message_count > 2` 且 `titleAi = false` 时跳过，避免无效 LLM 调用
- **Flyway 命名**：`V1_15__chat.sql`，一旦 apply 不可修改
- **MyBatis-Plus boolean**：`titleAi` 字段加 `@TableField("title_ai")` 显式映射
